# 공급망 DB 샘플 데이터 생성 가이드

## 1. 목적

본 문서는 공급망 DB를 구성하는 마스터 테이블 및 매핑 테이블의 샘플 데이터를 생성하기 위한 가이드입니다. 각 테이블의 컬럼 정의, 데이터 타입, 생성 규칙, 테이블 간 참조 관계(FK)를 명시하여, 실제 운영 데이터와 유사하면서도 일관성 있는 샘플 데이터를 만드는 것을 목표로 합니다.

향후 Ontology 구현 시 활용할 관계 정보(협력사-자재-소재-카테고리 간 연결)가 샘플 데이터에도 논리적으로 성립하도록 생성해야 합니다.

## 3. 테이블 개요

| 구분 | 테이블명 | 역할 |
|---|---|---|
| 마스터 | 협력사 마스터 | 협력사(법인) 정보 |
| 마스터 | 생산지(Site) 마스터 | 협력사 산하 생산 거점 정보 |
| 마스터 | 자재 마스터 | 자재 정보 |
| 마스터 | 소재 마스터 | 원소재 정보 |
| 마스터 | 카테고리 마스터 | 자재 분류 체계(LV1~LV4 계층) |
| 매핑 | 생산지-자재 매핑 | 어느 생산지가 어느 자재를 공급하는지 |
| 매핑 | 자재-소재 매핑 | 어느 자재가 어느 소재를 포함하는지 |
| 매핑 | 자재-카테고리 매핑 | 어느 자재가 어느 카테고리에 속하는지 |

**생성 순서 권장**: 마스터 테이블을 먼저 생성한 후, 마스터 테이블의 PK 값을 참조하여 매핑 테이블을 생성해야 합니다. (카테고리 마스터 → 자재 마스터 → 자재-카테고리 매핑 → 협력사 마스터 → 생산지 마스터 → 생산지-자재 매핑 → 소재 마스터 → 자재-소재 매핑 순서 권장)

## 4. 마스터 테이블 상세

### 4.1 협력사 마스터

| 컬럼명 | 타입 | 필수 | 생성 규칙 |
|---|---|---|---|
| No | INT | Y | 순번, 1부터 자동 증가 |
| 협력사 코드 (PK) | VARCHAR(10) | Y | 영문 2자 + 숫자 2~4자 조합 (예: BSY4, V676, GH78). 중복 불가 |
| 업체명(KOR) | VARCHAR(100) | Y | 한글 업체명. 가상의 업체명이어도 실제 업종에 어울리는 형태로 생성 |
| 업체명(ENG) | VARCHAR(100) | Y | 영문 업체명 (KOR과 동일 의미) |
| 기업식별코드 | VARCHAR(20) | Y | 국가별 사업자 식별 번호 형식 (예: 미국 EIN 형식 XX-XXXXXXX, 한국 사업자번호 형식 XXX-XX-XXXXX) |
| 국가/지역 | VARCHAR(50) | Y | 국가명 (예: United States, South Korea, Taiwan, Japan, China) |
| 행정구역 | VARCHAR(10) | Y | 국가별 행정구역 코드. 미국은 주(State) 코드 (예: PA, CA, TX). 타 국가는 해당 국가의 주/성/도 코드 또는 약어 사용 |
| 주소 | VARCHAR(200) | Y | 도로명 + 도시 + 행정구역 + 우편번호 형식의 가상 주소 |
| 위도 | DECIMAL(9,6) | Y | 행정구역 내 실제 위치 범위에 맞는 위도값 |
| 경도 | DECIMAL(9,6) | Y | 행정구역 내 실제 위치 범위에 맞는 경도값 |
| 활성여부 | BOOLEAN | Y | TRUE/FALSE. 대부분 TRUE, 일부(5~10%)는 FALSE로 비활성 케이스 포함 |
| 생성일 | DATE | Y | YYYY-MM-DD. 과거 1~5년 내 임의 날짜 |
| 수정일 | DATE | Y | 생성일 이후 날짜. 수정 이력이 없는 경우 생성일과 동일 |

**생성 시 주의사항**
- 위도/경도는 "행정구역" 값과 실제로 일치하는 지리적 범위 내에서 생성 (예: 행정구역이 PA면 펜실베이니아주 범위 내 좌표)
- 동일 국가/행정구역 내에서도 업종이 다양하게 분포하도록 구성 (반도체 소재, 화학, 전자자재 등 공급망 분석에 적합한 업종 위주)

---

### 4.2 생산지(Site) 마스터

| 컬럼명 | 타입 | 필수 | 생성 규칙 |
|---|---|---|---|
| No | INT | Y | 순번 |
| Site Code (PK) | VARCHAR(10) | Y | 형식: S + 5자리 숫자 (예: S00001) |
| 협력사 코드 (FK) | VARCHAR(10) | Y | 협력사 마스터의 협력사 코드 참조. 1개 협력사가 1~3개 Site를 갖도록 분포 |
| 생산지명 | VARCHAR(100) | Y | 예: "OO Plant 1", "OO 청주공장" 등 |
| 국가/지역 | VARCHAR(50) | Y | 협력사 본사와 다른 국가일 수 있음 (해외 생산법인 케이스 포함) |
| 행정구역 | VARCHAR(10) | Y | 위 국가/지역 기준 |
| 주소 | VARCHAR(200) | Y | 가상 주소 |
| 위도 | DECIMAL(9,6) | Y | 행정구역과 일치하는 좌표 |
| 경도 | DECIMAL(9,6) | Y | 행정구역과 일치하는 좌표 |
| 활성여부 | BOOLEAN | Y | TRUE/FALSE |
| 생성일 | DATE | Y | 협력사 생성일 이후 날짜 |
| 수정일 | DATE | Y | 생성일 이후 날짜 |

**생성 시 주의사항**
- 동일 자재 코드에 대해 리스크가 생산지별로 달라지는 케이스를 의도적으로 포함 (예: 같은 협력사의 미국 Site와 중국 Site가 같은 자재를 각각 생산)
- 전체 Site 중 일부는 협력사 본사와 동일 국가, 일부는 해외 생산기지로 분포

---

### 4.3 카테고리 마스터

| 컬럼명 | 타입 | 필수 | 생성 규칙 |
|---|---|---|---|
| No | INT | Y | 순번 |
| 카테고리 코드 (PK) | VARCHAR(10) | Y | 레벨을 식별 가능한 코드 체계 권장 (예: C1-001, C2-001 등) |
| 카테고리명 | VARCHAR(100) | Y | 예: TSP, TSP 원자재, 모듈/SSD 소자부품, 수동 |
| 레벨 | VARCHAR(5) | Y | LV1, LV2, LV3, LV4 중 하나 |
| 상위 카테고리 코드 (FK, 자기참조) | VARCHAR(10) | N | LV1은 NULL. LV2 이상은 상위 레벨의 카테고리 코드를 참조 |
| 활성여부 | BOOLEAN | Y | TRUE/FALSE |
| 생성일 | DATE | Y | |
| 수정일 | DATE | Y | |

**생성 시 주의사항**
- 계층 구조가 끊기지 않도록 LV1부터 순서대로 생성 (LV2 생성 시 반드시 존재하는 LV1 코드를 상위 카테고리로 지정)
- 예시 체계: LV1(TSP) → LV2(TSP 원자재) → LV3(모듈/SSD 소자부품) → LV4(수동)

---

### 4.4 자재 마스터

| 컬럼명 | 타입 | 필수 | 생성 규칙 |
|---|---|---|---|
| No | INT | Y | 순번 |
| 자재 코드 (PK) | VARCHAR(6) | Y | 영문/숫자 조합 4~6자리 |
| 자재명(KOR) | VARCHAR(100) | Y | 한글 자재명 |
| 자재명(ENG) | VARCHAR(100) | Y | 영문 자재명 |
| 자재유형 | VARCHAR(50) | Y | 예: 능동소자, 수동소자, 모듈, 원자재 등 |
| 활성여부 | BOOLEAN | Y | TRUE/FALSE |
| 생성일 | DATE | Y | |
| 수정일 | DATE | Y | |

**생성 시 주의사항**
- 자재유형은 카테고리 마스터의 레벨 분류와 의미상 정합성을 가지도록 구성

| 활성여부 | BOOLEAN | Y | TRUE/FALSE |
| 생성일 | DATE | Y | |

## 5. 매핑 테이블 상세

### 5.1 생산지-자재 매핑

| 컬럼명 | 타입 | 필수 | 생성 규칙 |
|---|---|---|---|
| No | INT | Y | 순번 |
| Site Code (FK) | VARCHAR(10) | Y | 생산지 마스터 참조 |
| 자재 코드 (FK) | VARCHAR(6) | Y | 자재 마스터 참조 |
| 공급 비중(%) | DECIMAL(5,2) | N | 0~100 사이 값. 동일 자재를 여러 Site가 공급하는 경우 합이 100%에 가깝도록 분배 (필수 아님, 정보 없을 시 NULL) |
| 주공급여부 | BOOLEAN | Y | 해당 Site가 해당 자재의 주공급처인지 여부 |
| 활성여부 | BOOLEAN | Y | TRUE/FALSE |
| 생성일 | DATE | Y | |
| 수정일 | DATE | Y | |

**생성 시 주의사항 (핵심)**
- **동일 자재 코드가 서로 다른 Site Code와 여러 번 매핑되는 케이스를 반드시 포함**해야 합니다. (예: 자재 코드 A001이 미국 Site와 중국 Site에서 각각 생산되는 경우, 2개 행으로 표현)
- 이를 통해 "동일 자재라도 생산 위치에 따라 리스크가 달라지는" 분석 시나리오를 샘플 데이터에서 검증할 수 있어야 합니다.

---

### 5.2 자재-소재 매핑

| 컬럼명 | 타입 | 필수 | 생성 규칙 |
|---|---|---|---|
| No | INT | Y | 순번 |
| 자재 코드 (FK) | VARCHAR(6) | Y | 자재 마스터 참조 |
| 소재 코드 (FK) | VARCHAR(10) | Y | 소재 마스터 참조 |
| 포함 비중(%) | DECIMAL(5,2) | N | 해당 자재 내 소재의 비중. 정보 없을 시 NULL |
| 포함여부 | BOOLEAN | Y | TRUE/FALSE (비중 정보가 없을 경우 최소한 포함 여부는 표시) |
| 활성여부 | BOOLEAN | Y | TRUE/FALSE |
| 생성일 | DATE | Y | |
| 수정일 | DATE | Y | |

**생성 시 주의사항**
- 1개 자재가 여러 소재를 포함하는 1:N 관계가 자연스럽게 나타나도록 구성 (자재 1개당 평균 2~5개 소재 매핑 권장)

---

### 5.3 자재-카테고리 매핑

| 컬럼명 | 타입 | 필수 | 생성 규칙 |
|---|---|---|---|
| No | INT | Y | 순번 |
| 자재 코드 (FK) | VARCHAR(6) | Y | 자재 마스터 참조 |
| 카테고리 코드 (FK) | VARCHAR(10) | Y | 카테고리 마스터 참조. 일반적으로 최하위 레벨(LV4 또는 가장 세부 레벨) 카테고리를 매핑 |
| 생성일 | DATE | Y | |
| 수정일 | DATE | Y | |

**생성 시 주의사항**
- 1개 자재는 원칙적으로 1개의 최하위 카테고리에만 속하도록 구성 (다중 분류가 필요한 경우는 별도 협의)

## 6. 전체 데이터 일관성 체크리스트

샘플 데이터 생성 후 아래 항목을 검증합니다.

- [ ] 모든 매핑 테이블의 FK 값이 해당 마스터 테이블에 실제로 존재하는가
- [ ] 동일 자재 코드가 2개 이상의 Site Code와 매핑된 케이스가 최소 1건 이상 존재하는가 (생산지별 리스크 차이 시나리오 검증용)
- [ ] 카테고리 마스터의 상위 카테고리 코드가 끊기지 않고 LV1까지 연결되는가
- [ ] 활성여부가 FALSE인 협력사/자재/소재가 매핑 테이블에서도 적절히 비활성 처리되어 있는가 (또는 비활성 상태가 매핑에 미치는 영향에 대한 별도 규칙 확인 필요)
- [ ] 위도/경도 값이 명시된 행정구역의 실제 지리적 범위와 일치하는가
- [ ] 생성일/수정일의 선후관계가 마스터-매핑 테이블 간 논리적으로 맞는가 (매핑 테이블의 생성일은 참조하는 마스터 데이터의 생성일 이후여야 함)

## 7. 샘플 데이터 규모 권장

| 테이블 | 권장 건수 (초기 PoC 기준) |
|---|---|
| 협력사 마스터 | 20~30건 |
| 생산지 마스터 | 30~50건 (협력사당 1~3개) |
| 카테고리 마스터 | 42건 (9개 대분류 기준, 추후 확장) |
| 자재 마스터 | 50~100건 |
| 소재 마스터 | 30~50건 |
| 생산지-자재 매핑 | 60~150건 |
| 자재-소재 매핑 | 100~300건 |
| 자재-카테고리 매핑 | 50~100건 (자재 마스터 건수와 동일) |
