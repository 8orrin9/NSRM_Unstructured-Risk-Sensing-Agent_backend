# Insight KG 기반 반도체 공급망 Scenario Group 생성 설계 문서

- 프로젝트: 삼성전자 DS N-SRm 고도화 프로젝트 (AI Task)
- 문서 버전: v0.1 (초안)
- 작성 목적: Claude Code를 통한 구축 요청 시 전달할 설계 근거 문서

---

## 1. 프로젝트 개요

- **목적**: 전문 분석기관(코트라 등) 리포트 기반 Insight KG를 구축하고, 별도 수집된 뉴스를 이 KG에 매핑하여 뉴스 간 관계성을 파악, 복수 뉴스가 결합된 **Scenario Group**을 생성한다.
- **핵심 설계 원칙**: 뉴스 개별 건이 아닌, 여러 뉴스가 KG상에서 서로 연결될 때에만 하나의 Scenario Group으로 묶는다.
- **범위 경계 (Scope Boundary)**
  - 본 파이프라인은 **Scenario Group 생성까지가 목표**이며, Risk 여부 판단은 후속 Agent의 역할이다.
    - 후속 Agent 처리 흐름(참고): Scenario Group의 주요 키워드 추출 → 기 보유 공급망 DB에서 관련 엔티티 검색 → Risk 발생 가능성 판단
  - 소스 신뢰도 가중치 부여, 그룹화 유의성에 대한 정량 평가 지표 설계는 본 파이프라인 범위에서 **제외**한다.

---

## 2. 전체 아키텍처 (Two-Layer Architecture)

| 구분 | Layer 1 — Insight KG | Layer 2 — News Mapping |
|---|---|---|
| 소스 | 코트라 등 전문 분석기관 리포트 | 기 설정된 Risk Factor-Keyword 기반 수집 뉴스 |
| 성격 | 정적, 구조적 (원인-결과, 공급망 위계) | 동적, 이벤트성, 타임스탬프 존재 |
| 역할 | 리스크가 흐를 수 있는 관계 구조 | 그 구조 위에서 발생하는 사건의 흐름 |
| 갱신주기 | 주 1회 | 시간당 1회 (PoC 단계: 최초 1회 매핑 후 재사용, 필요시 재매핑) |

- 두 레이어는 별개로 유지되며, Layer 2는 Layer 1을 **참조만** 하고 KG 자체를 변형하지 않는다.
- 뉴스 자체끼리 직접 비교하는 것이 아니라, **Layer 1 KG상에서 공유하는 노드/경로를 매개로** 뉴스 간 관계를 도출하는 것이 본 설계의 핵심 로직이다.

## 3. Phase 1 — Insight KG 구축

- **엔진**: LightRAG
- **스토리지**: LightRAG 기본(자체) 스토리지 활용
  - 구성: 그래프(NetworkX) + KV(JSON) + 벡터(NanoVectorDB), 파일 기반으로 영속화되어 프로세스 종료 시에도 디스크에 데이터가 남음
  - PoC 규모(주 1회 갱신)에서는 적합하나, 향후 데이터 규모 확장 시 Neo4j/외부 벡터DB(Qdrant, Milvus 등)로의 마이그레이션을 고려할 것 (동시성 처리, 대용량 확장성, 그래프 질의 기능, 백업/버전관리 측면에서 기본 스토리지는 운영 환경에는 권장되지 않음)
- **엔티티 타입 스키마**
  - Risk Factor-Keyword Set(카테고리 및 세부 키워드 체계)을 기반으로 정의 예정
  - **[Open Issue]** 구체적인 엔티티 타입 셋은 Risk Factor-Keyword Set 확보 후 확정 필요
- **엔티티 정규화**
  - 동일 엔티티의 표기 이형(예: 기업 정식명/약칭/영문명 등)을 통일하는 공통 룩업 테이블을 구성
  - 이 룩업 테이블은 Phase 1(KG 구축)과 Phase 2(뉴스 매핑) 양쪽에 공통으로 적용하여 일관성 확보
- **관계(엣지) 타입 태깅**
  - 엔티티 추출과 동시에 관계 카테고리(예: 인과/정책규제/공급/지리/서술 등) 분류 태그를 부여
  - **[Open Issue]** 구체적인 관계 카테고리 taxonomy는 Risk Factor-Keyword Set 확보 후 확정 필요

## 4. Phase 2 — 뉴스 → KG 매핑 (Grounding)

- **매핑 방식**: 임베딩 유사도 기반 매칭 (LightRAG의 벡터 인덱싱 구조와 자연스럽게 결합)
- **전처리 파이프라인 (뉴스 원문 직접 매핑이 아닌 2단계 가공 적용)**
  1. **뉴스 요약**: 뉴스에서 핵심 엔티티·이벤트·팩트만 추출한 구조화 요약 생성 (기사 문체의 노이즈 제거)
  2. **HyDE 적용**: 요약을 기반으로 "전문 분석기관 리포트의 한 문단이었다면 어떻게 서술되었을지"를 가정한 가상 문서(Hypothetical Document)를 LLM으로 생성
     - HyDE 생성 프롬프트에는 Insight KG 원본 리포트의 문체 샘플을 few-shot으로 포함하여, 실제 KG 노드 설명과의 문체 격차를 최소화
  3. **임베딩 및 매칭**: 뉴스 원문이 아닌 HyDE 가상 문서를 임베딩하여 KG 노드 임베딩과 유사도 비교, 매핑 확정
- **매핑 신뢰도 임계치**: 일정 유사도 스코어 이상인 경우에만 매핑을 확정하고, 하위 매핑 결과는 이후 관계성 판단에서 배제
- **산출물**: 뉴스별 "매핑된 KG 노드/엣지 리스트 + 매핑 신뢰도 점수"

## 5. Phase 3 — 뉴스 간 관계성 파악

- **판단 기준 (우선순위 순)**
  - **① 경로 인접형 (주요 근거)**: 서로 다른 뉴스가 매핑된 KG 노드 간 경로 거리가 2-hop 이내인 경우
  - **② 노드 공유형 (보조 근거)**: 서로 다른 뉴스가 동일 KG 노드에 매핑된 경우
    - 단독으로는 관계 인정 불가, 반드시 경로 인접형 근거와 결합되었을 때에만 보조 신호로 활용
  - **시간 근접형**: 본 설계에서 배제 (이벤트 시차 기반 판단은 복잡도 및 허위 상관 위험이 커서 채택하지 않음)
- **허위 상관(Spurious Correlation) 방지 로직**
  - **경로 길이 제한**: KG 경로 거리가 2-hop을 초과하면 관계로 인정하지 않음
  - **엣지 특이성 기반 필터링 (허브 노드 페널티 대체)**
    - 기존 검토안(노드 차수 기반 일괄 페널티)은 "미국–반도체–중국"과 같이 차수가 높더라도 실제로는 강한 연관성(수출규제 등)을 갖는 구조를 놓칠 위험이 있어 폐기
    - 대체 기준: 노드의 차수가 아닌 **경유하는 엣지의 성격**으로 신뢰도를 판단
      - 서술/소속형 엣지(예: 단순 언급, 소속 관계) 경유 시 → 신뢰도 하향
      - 인과/정책/공급형 엣지(예: 수출규제, 의존관계, 공급관계) 경유 시 → 노드 차수와 무관하게 신뢰도 유지
    - **[Open Issue]** 구체적인 엣지 카테고리 구분 기준 및 신뢰도 가중치는 Risk Factor-Keyword Set 확보 후 확정 필요
    - 구현 방향(합의됨): ① KG 추출 시점에 관계 카테고리 태깅 → ② 추출 후 LLM 기반 재분류 후처리로 정확도 보완 → ③ 경로 탐색 시점에 이 카테고리 라벨을 필터/가중치 기준으로 적용
  - **매핑 신뢰도 임계치 적용**: Phase 2에서 설정한 임계치 미만으로 매핑된 노드는 관계성 판단에서 배제
  - **노드 공유형 단독 배제**: 경로 인접형 근거 없이 노드 공유형만으로 이루어진 관계는 인정하지 않음
- **산출물**: 뉴스-뉴스 관계 그래프 (Layer 2 파생 그래프, Layer 1과는 별개)

## 6. Phase 4 — Scenario Group 생성

- **클러스터링**: Phase 3에서 생성된 뉴스-뉴스 관계 그래프에 그래프 클러스터링 알고리즘(예: Louvain, Leiden) 적용
  - LightRAG는 기본적으로 커뮤니티 탐지(Leiden 등)를 제공하지 않으므로, 본 클러스터링 단계는 별도 구현이 필요함
- **그룹 승격 조건 (Scenario Group으로 인정하는 기준)**
  - 최소 뉴스 수: 군집 내 뉴스 2건 이상
  - 최소 연결 근거: 군집을 구성하는 뉴스 쌍 중 최소 1개 이상이 경로 인접형 근거로 연결되어 있을 것
- **산출물**: 각 Scenario Group에 대해 반도체 공급망 관점의 인과 서사(narrative)를 LLM으로 생성
  - 명칭은 "Risk Scenario Group"이 아닌 **"Scenario Group"**으로 통일 (Risk 여부는 후속 Agent가 판단하므로 본 단계 산출물에는 Risk 판단이 포함되지 않음)
  - 스코어링(리스크 점수화)은 수행하지 않음

## 8. Agent_3 다중 시나리오 분할 + KG 경로 활용 (향후 확장)

### 8.1 배경
Phase 2~3 재설계 결과, **뉴스 그룹화가 엔티티 문자열 매칭 기반으로 단순화**되었습니다.
- Phase 2: 뉴스 → LLM 엔티티 추출 → 문자열 매칭
- Phase 3: 공유 엔티티 2개 이상 → 그룹화 → 통합 문서

이로 인해 **하나의 그룹에 여러 주제가 섞일 수 있음**:
- 예: 그룹 내 뉴스A (중국 희토류 규제), 뉴스B (중국 반도체 수입), 뉴스C (중국 통화 정책)
- Agent_2 태그: [RAW_RARE_EARTH, SUPPLIER_CHINA, MATERIAL_SEMICONDUCTOR, EVENT_CURRENCY]
- 문제: 주제가 섞여 DB 검색 쿼리 품질 저하

### 8.2 해결책: Agent_3에서 시나리오 분할 + KG 경로 활용

**핵심 아이디어**:
- Agent_2 태그를 주제별로 클러스터링 → 여러 시나리오 생성
- 각 시나리오 태그 → Insight KG 엔티티 매핑
- KG 경로 탐색 (2-hop 이내)
- 경로 기반 "왜 이 태그들이 연관되는가" 설명 생성

**처리 흐름**:
```
Agent_2 태그 출력
    ↓
[Agent_3: generate_risk_scenario 노드]
    ↓
1. 태그 클러스터링 (온톨로지 기반)
   → 시나리오 1: [RAW_RARE_EARTH, SUPPLIER_CHINA, POLICY_EXPORT_CONTROL]
   → 시나리오 2: [MATERIAL_SEMICONDUCTOR, SUPPLIER_CHINA]
   → 시나리오 3: [EVENT_CURRENCY, SUPPLIER_CHINA]
    ↓
2. 각 시나리오별 처리:
   (2-1) 태그 → KG 엔티티 매핑
         RAW_RARE_EARTH → "희토류"
         SUPPLIER_CHINA → "중국"
         POLICY_EXPORT_CONTROL → "수출 규제"
    ↓
   (2-2) KG 경로 탐색 (max_distance=2)
         중국 --[POLICY_REGULATION]--> 희토류 --[SUPPLY]--> 반도체
    ↓
   (2-3) 경로 기반 시나리오 설명 생성 (LLM)
         "중국이 희토류 수출을 규제하면(POLICY_REGULATION), 
          희토류가 반도체 생산에 필수적이므로(SUPPLY), 
          반도체 공급망에 차질이 예상됩니다."
    ↓
   (2-4) DB 검색 쿼리 생성 (경로상 중간 엔티티 포함)
         SELECT * FROM supplier WHERE material_type IN ('희토류', '반도체')
         AND country='중국'
    ↓
3. 각 시나리오 → Agent_4로 전달
```

### 8.3 KG 역할

**현재 (Phase 2~3)**:
- 엔티티 목록 (509개) + 정규화 테이블 (39 mappings) 사용
- 관계(422 edges) 및 경로 정보는 **사용 안 함**

**향후 (Agent_3 확장)**:
- **태그 → KG 엔티티 매핑**: 정규화 테이블 + 509개 노드 활용
- **KG 경로 탐색**: 422개 엣지 활용 (2-hop 이내)
- **관계 카테고리 활용**: POLICY_REGULATION, SUPPLY, CAUSAL 등 엣지 타입으로 시나리오 설명 강화
- **엣지 특이성 필터링**: HIGH specificity (88.7%) 엣지만 신뢰, MEDIUM/LOW는 보조

**구현 파일**:
- `dev/Agent_3_DB_Searcher/nodes/generate_risk_scenario.py` (수정)
- `dev/Agent_3_DB_Searcher/utils/kg_path_finder.py` (신규)
- `dev/Agent_3_DB_Searcher/utils/tag_kg_mapper.py` (신규)

**비용**:
- KG 경로 탐색: 무료 (NetworkX)
- 시나리오 설명 생성: ~$0.01/시나리오 (gpt-4o-mini)
- 총: ~$0.05/뉴스 그룹 (5개 시나리오 가정)

### 8.4 기대 효과

1. **다중 시나리오 품질 향상**: 희토류/반도체/통화 → 각각 독립 시나리오
2. **DB 검색 정확도 향상**: 경로상 중간 엔티티를 검색 키워드로 추가
3. **Risk 분석 설명력 강화**: KG 경로 근거를 Agent_4로 전달
4. **KG 구조 활용**: 이미 구축된 KG (422 edges, $11.42)의 가치 극대화
