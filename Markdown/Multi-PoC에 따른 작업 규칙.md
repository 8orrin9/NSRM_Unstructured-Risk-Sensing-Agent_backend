## 범위 규칙
- 수정 가능 영역: `/poc-a`, 그리고 (확장 목적의) `/shared`
- **`/poc-b`는 절대 수정하지 않는다.**
- `/shared`를 수정해야 하면: 기존 코드 변경보다 확장을 우선하고, poc-b가 깨지지 않는지 확인한다. 공용 변경은 `feature/shared/*` 브랜치로 분리한다.

## 브랜치
- 작업 브랜치: `feature/poc-a/<기능>` (항상 `develop`에서 분기)
- beta로 push해 테스트 → PR → develop merge
